# פרויקט יצירת ספרי Moodle - מעבדה חשמל 1 (EELab1 Moodle Book)

פרויקט זה מיועד ליצירה, ניהול ואריזה אוטומטית של חומרי לימוד אינטראקטיביים עבור מערכת Moodle (בפורמט Moodle Book) במכללה. הפרויקט לוקח קבצי HTML גולמיים, מזריק אליהם עיצובים (CSS), ומייצר קבצי ZIP המוכנים לייבוא ישיר לתוך המודל.

## 📂 מבנה הפרויקט (Architecture)

הפרויקט בנוי בצורה מודולרית ומחולק לתיקיות הבאות:

* **`src/content/`**: תיקיית המקור. כאן יושבים כל קבצי ה-HTML הגולמיים של הניסויים השונים (`Exp01`, `Exp02` וכו').
* **`src/templates/`**: תבניות העיצוב. מכיל את הקובץ `styles.css` שאחראי על הנראות הכללית, ואת תבניות המסגרת ל-Moodle.
* **`scripts/`**: מנוע הבנייה (Build Pipeline). מכיל סקריפטים ב-Python ו-Node.js שאחראים על אוטומציית האריזה.
* **`dist/`**: תיקיית התוצרים (נוצרת אוטומטית).
  * `dist/previews/` - תצוגה מקדימה של דפי ה-HTML (מאפשר לראות איך הדף ייראה במודל לפני ההעלאה).
  * `dist/release/` - קבצי ה-ZIP המוכנים לייבוא ל-Moodle.

## 🚀 איך מריצים ואיך בונים (Build Instructions)

כדי לארוז מחדש את כל הניסויים וליצור את קבצי ה-ZIP, אין צורך להריץ סקריפטים בנפרד. יש פקודת "קסם" אחת שעושה הכל.

1. פתח טרמינל (CMD או Git Bash) בתיקיית הפרויקט.
2. ודא שהתקנת ספריות בפעם הראשונה (על ידי הפקודת `npm install`).
3. הרץ את הפקודה הבאה:
   ```bash
   npm run release
   ```
   **מה הפקודה הזו עושה?**
   - **מנקה (`clean`)**: מוחקת את תיקיית `dist` הישנה כדי למנוע כפילויות.
   - **בונה (`build`)**: לוקחת את העיצוב מ-`styles.css` ומזריקה אותו (Inline) לתוך קבצי ה-HTML.
   - **אורזת (`package`)**: מפעילה את פייתון שסורק את כל התיקיות ומייצר את קבצי ה-ZIP הסופיים.

*(במידה ויש שגיאת הרשאות ב-Windows שמקורו ב-PowerShell, ניתן פשוט להריץ `python scripts/rebuild_and_clean_dist.py`)*.

## 📝 איך מוסיפים דף או ניסוי חדש?

מערכת האריזה היא **דינמית ואוטומטית**. כדי להוסיף תוכן:
1. צור או הוסף קובץ HTML חדש בתיקיית התוכן המתאימה (למשל: `src/content/EELab1/Exp01/moodle_book_import_v2/`).
2. תן לקובץ שם באנגלית (הסדר האלפביתי של שמות הקבצים קובע את סדר הפרקים במודל).
3. הרץ `npm run release`.
4. זהו! המערכת תזהה את הקובץ החדש ותכניס אותו ל-ZIP הרלוונטי באופן אוטומטי.

## ⚙️ טכנולוגיות מרכזיות
* **Node.js + Juice**: משמש להזרקת קוד CSS לתוך ה-HTML כדי ש-Moodle לא ידרוס את העיצוב (Inline Styles).
* **Python**: משמש לניהול קבצים, יצירת ארכיוני ZIP, וארגון התיקיות לפני שחרור גרסה.
* **GitHub Actions**: תהליך ה-Build רץ אוטומטית בענן בכל פעם שעושים Push ל-Branch הראשי.
